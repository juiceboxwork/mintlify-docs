## **Agent Overview**

Juicebox Agents are automated sourcing assistants that continuously source for talent based on the criteria you define. Instead of running the same search manually every day, Agents work in the background to monitor the market and surface new, relevant candidates as they appear.

By setting up an Agent, you can:

- **Calibrate naturally** through conversation and profile feedback that aligns the Agent with your hiring preferences
- **Autonomously search** based on your calibrated role, location, and team requirements to surface **stronger candidates faster**
- **Continuously refine results** using learnings from approved and rejected profiles to improve candidate quality over time
- **Automatically send outreach sequences** from a connected mailbox with customizable messaging and timing (optional based on the Agent’s configuration)

<Note>
  Note: Agents are an optional add-on to your Juicebox subscription.
</Note>

### Use Cases

To keep Agents running and drive the most ROI, Agents are best utilized for roles that are repeatable, evergreen, or high-volume. These use cases include but are not limited to:

1. **Streamlined job titles and backgrounds that are consistent across companies / industries**
   1. Examples: Find Technical Recruiters in the Bay Area; Find Account Executives in NYC
2. **Rediscovering past ATS talent**
   1. Examples: Find candidates from my ATS who have 3\+ yoe in LLMs; Find all silver-medalists or candidates who made it to X stage
3. **Market mapping and competitive intelligence**
   1. Examples: Find VPs or Directors at X competitors; Find all SWEs who just went through a layoff; Find any candidate skilled in AI/ML within the Bay Area
4. **Business development or hyper-targeted prospect lists**
   1. Examples: Find Operation leads at emerging companies; Find VPs of Talent in Healthtech at a company of 200 or less

## **Setting up an Agent**

{/* TODO [agents-restructure]: Reinforce that agents ONLY exist inside projects — there is no standalone "create an agent" entry point anymore. Creating an agent = creating a project. */}

Every Agent in Juicebox lives inside a project — creating an Agent means creating (or opening) the project it will source for. To set up an Agent, please follow the below steps:

1. Navigate to **Projects** and click **Create new project**, or open an existing project you'd like to add an Agent to.
2. Name the project after the role, configure collaborators, and link it to your ATS job (business plans only).
   {/* TODO [agents-restructure]: Call out the ATS-job-launched flow — if a user links an ATS job and launches with a JD at project creation, the project homepage is bypassed entirely and the Search/Agent toggle in step 3 does not appear. Document how to get back to picking Agent from that entry path (e.g. via the project's table view / toggle). */}
3. From the project homepage, select **Agent** to hand this role off to an Agent instead of running a manual search. If you're already inside a project running a manual search, you can toggle over to **Agent** at any time.
   {/* TODO [agents-restructure]: Note that the project homepage is the only place the Search vs Agent choice is made now — the old standalone "create agent" flow is gone. */}

![Screenshot 2026-05-18 at 3.56.19 PM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-18_at_3-56-19_PM.png)

<Tip>
  For business plan customers, link the project directly to your ATS job and enable auto-exports to keep a centralized source of truth across systems.
</Tip>

1. Tell the Agent what you are looking for by utilizing one of the four methods below:
   1. Describe the type of candidate using your natural language
      1. **Example:** I'm looking for a senior product designer with 5\+ years of experience in B2B SaaS, based in SF or NYC, who has worked on design systems
   2. Input a link to the Job Description for the Agent to source against
      1. **Example:** Find candidates for this role: \[insert public JD link\]
   3. Replicate a candidate profile by inputting the candidate’s LinkedIn URL
      1. **Example:** Find similar candidates to \[insert candidate’s LinkedIn URL\]
   4. Copy the job description or an intake meeting’s transcript for the Agent to analyze and parse
   ![Screenshot 2026-05-19 at 3.21.41 PM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-19_at_3-21-41_PM.png)

Once submitted, the Agent will review your request to automatically configure the initial filters and search criteria. Depending on the request, your Agent may provide you with its top picks to review or ask follow-up questions for additional context, similar to how you would typically collaborate with a recruiter.

![Screenshot 2026-05-19 at 6.04.41 PM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-19_at_6-04-41_PM.png)

1. Review your filters and criteria to confirm that all requirements are included, or provide the Agent with any additional details

![image](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/image.png)

1. Review the selected profiles to provide feedback and further refine your results

![Screenshot 2026-05-18 at 4.36.17 PM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-18_at_4-36-17_PM.png)

If a profile has the right experience and skills, select Looks Good. If the candidate is misaligned, select Give Feedback, and leave any reasons why the candidate is not a fit (i.e., average tenure should be at least 3 years) within the feedback pop up.

![Screenshot 2026-05-19 at 6.07.57 PM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-19_at_6-07-57_PM.png)

<Warning>
  Please note, approval/rejection feedback is optional but **highly encouraged** as the Agent will take these learnings (good or bad) into consideration for the next batch of profiles provided. In order to move on to the next step, we also recommend reviewing profiles until you can approve 3 candidates in a row.
</Warning>

1. Continue to configure the Agent based on your workflow by selecting **Outreach Settings**
   ![Screenshot 2026-05-18 at 4.48.56 PM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/0667fabc-9251-45f4-bf19-a13a153315d1.png)
2. From here, you will move on to finalize the Agent’s settings, where you can configure how many leads the Agent sources each day and choose whether the Agent should automatically contact those leads or simply send them for further review.

The first setting available is your **daily sourcing target**, where you can choose how many leads you would like the Agent to source each day. Agents can deliver up to 25 leads per day.

![Screenshot 2026-05-20 at 8.42.42 AM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-20_at_8-42-42_AM.png)

<Tip>
  Agents are always running and do not take weekends or PTO. The number of leads selected should represent how many leads you/your team have time to review on a daily or weekly basis.
</Tip>

After selecting your daily lead target, click **Next, outreach settings** to configure how your Agent will handle its sourced candidates.

![Screenshot 2026-05-20 at 8.54.07 AM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-20_at_8-54-07_AM.png)

Please note that Agents will only contact candidates using a sequence that has been created and reviewed by you. During setup, you can choose from the following options:

- **Use Existing Sequence** to select a previously created sequence
- **Generate AI Sequence** to create a personalized outreach sequence based on your conversation with the Agent and the role requirements
- **Add to Shortlist** if you would prefer to review and approve candidates before any outreach is sent

After you’ve selected your outreach settings, click **Next, approval type** to control how your leads are processed after sourcing.

![Screenshot 2026-05-20 at 8.56.10 AM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-20_at_8-56-10_AM.png)

Here, there are two options available which will perform the following actions based on your workflow:

- **Automatic,** if you’d like the Agent to shortlist leads on your behalf
- **Manual,** if you’d like to continue approving/rejecting candidates to help the Agent capture additional learnings

Once all settings are finalized, click **Start sourcing** to activate the Agent and collect your first batch of leads!

![Screenshot 2026-05-20 at 9.00.21 AM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-20_at_9-00-21_AM.png)

<Tip>
  Agent settings can be updated at any time from the Agent’s **Settings** tab. For example, if you initially choose to manually review your sourced leads, you can later switch to automatic outreach at any point, and vice versa.
</Tip>

## Reviewing Leads

Once your Agent has sourced its daily batch of leads, you can open the Agent to review both pending and approved candidates. To review leads awaiting approval, click **Review Leads** in the top-right corner or select a candidate’s name directly to open their profile and evaluate their fit.

![Screenshot 2026-05-18 at 5.07.51 PM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-18_at_5-07-51_PM.png)

During the review process, you can leave additional feedback to help the Agent continue learning and improve the quality of future leads. All feedback submitted during reviews is also stored within the **Calibrate** chat, allowing you to reference the Agent’s full history and decision-making context in one place.

![Screenshot 2026-05-19 at 6.28.14 PM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-19_at_6-28-14_PM.png)

If you’d like to revisit any previously approved profiles, please select **Profiles Approved** from the top navigation bar within the **Sourcing** tab. From there, you can review each candidate’s match summary or open their Juicebox profile directly.

![Screenshot 2026-05-18 at 5.14.07 PM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-18_at_5-14-07_PM.png)

## Contacting Leads

Just like our Projects, each Agent will have a dedicated shortlist where you can manage candidate outreach and track candidate statuses.

![Screenshot 2026-05-18 at 5.16.54 PM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-18_at_5-16-54_PM.png)

Whether your Agent is configured for manual review or automatic outreach, the shortlist can help you select candidates to share, export, or follow up with, similar to how you would from your Project’s shortlist. For more information on shortlists, please view our help article [here](/shortlist-profiles).

## Viewing Insights

Similar to projects, Agents also include an Insights report that provides a breakdown of the Agent’s leads by location, job title, skills, years of experience, and more. To access your Agent’s Insights report, select **View Insights** from the right-hand side of the Agent’s **Calibrate** screen.

## Switching an existing project to an Agent

Already have a project running a manual search that seems like a strong fit for an Agent? From your project, toggle over to **Agent** to hand the role off to an Agent using your existing filters and search criteria as a starting point.

![Screenshot 2026-05-20 at 9.09.07 AM](https://raw.githubusercontent.com/juiceboxwork/docs/kb-migration/images/juicebox-agents/Screenshot_2026-05-20_at_9-09-07_AM.png)

The Agent lives inside the same project as your search, so any leads it surfaces and any candidates you shortlist stay together with your manual work. You can toggle back to manual search at any time.

## Glossary

{/* TODO [agents-restructure]: These statuses now surface on the redesigned Projects page (with a "Show agents only" filter and status filters), not on a standalone Agents page. Update surrounding context so users know where to find them. */}

Understand the key terms associated to an Agent and its current status:

- **Sourcing:** An Agent that is actively sourcing or contacting leads, and takes up a seat.
- **Paused:** An Agent that is paused manually or due to four days of inactivity, and takes up a seat.
- **Out of Leads:** An Agent that has finished running and should either be recalibrated or closed, and takes up a seat.
- **Calibrating:** An Agent that is being set up or calibrated, but has not yet started sourcing, and does **not** take up a seat.
- **Closed:** An Agent that is turned off and cannot be restarted, and does **not** take up a seat.
